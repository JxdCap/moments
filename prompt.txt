基于当前项目实际状态执行本任务。先阅读并理解以下文件后再动手：src/services/demoData.ts、src/services/posts.ts、src/hooks/usePosts.ts、src/types/moment.ts、src/components/MomentCard.tsx、src/components/MomentCard.module.css。确认当前项目仍以本地 demo 数据为主展示，再执行本任务。

本轮目标非常单一：**为动态增加轻量位置字段展示，并补充对应 demo 数据。**  
不要改照片模式，不要改标签筛选，不要改评论点赞逻辑，不要改动画，不要接 PocketBase，不要顺手重构，不要扩展无关功能。

一、必须完成的内容

1. 为动态增加位置字段
在当前项目的数据结构中，增加一个轻量位置字段，用于表示动态发布地点。
字段命名保持简洁清晰，例如 `location`，类型使用可选字符串即可。
要求：
- 不要引入复杂定位对象
- 不要做经纬度
- 不要做地图卡片
- 只支持轻量文本位置展示，例如：
  - 深圳·南山
  - 上海·武康路
  - 西湖边
  - 公司楼下

2. 数据层同步兼容
在不接 PocketBase 的前提下，先打通本地 demo 数据链路。
必要时同步更新：
- src/types/moment.ts
- src/services/demoData.ts
- src/services/posts.ts
但只做和 location 字段直接相关的最小改动。
不要顺手改别的字段，不要重构 posts 服务。

3. 卡片中展示位置
在动态卡片中，把位置展示在一个合理且克制的位置：
- 放在 tags 下方
- 放在底部时间 / 来源 / 三点菜单上方
- 若当前动态没有 location，则不显示任何占位
视觉要求：
- 轻量
- 不抢正文和图片
- 更像朋友圈里的补充信息，不像正文的一部分
- 可配一个很轻的位置图标或纯文本前缀，但不要做重图标风格

4. 补充 demo 数据
在现有 demoPosts 基础上，补充若干带 location 的示例，让页面中至少能看到：
- 纯文字 + location
- 单图 + location
- 文字 + 单图 + location
- 文字 + 多图 + location
位置文案尽量自然，不要所有动态都带 location，避免显得假。

二、实现约束

1. 不要改 PostType 类型体系
2. 不要新增页面或模式
3. 不要改照片模式逻辑
4. 不要改灯箱逻辑
5. 不要改评论、点赞、作者模式逻辑
6. 不要引入新依赖
7. 不要做地图、经纬度、逆地理编码等复杂功能
8. 不要为了 location 去重构整个数据模型

三、显示要求

1. location 在卡片中应明显可见，但存在感低于正文、图片和 tags
2. 没有 location 的动态完全保持现状，不要出现空白占位
3. 页面里至少能看到 3～4 条带 location 的 demo 动态
4. 如果现有 demo 排序导致新增示例过于靠后，可适度调整发布时间，但不要破坏现有置顶逻辑

四、完成后要求

1. 至少执行一次可验证检查，如 npm run build
2. 输出时明确汇报：
- location 字段具体加在了哪些文件
- demo 中新增了多少条带 location 的动态
- 卡片里 location 放在什么位置展示
- 修改了哪些文件
- 构建结果

重点提醒：本轮不是做复杂定位系统，而是“增加一个轻量 location 字段，让页面更像真实朋友圈”。
