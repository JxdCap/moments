基于当前项目实际状态执行本任务。先阅读并理解以下文件后再动手：src/components/Timeline.tsx、src/components/MomentCard.tsx、src/components/MomentCard.module.css、src/services/demoData.ts、src/services/posts.ts、src/types/moment.ts。确认当前项目已经有“全部 / 照片”两种浏览模式、并且已有 video 类型动态后，再执行本任务。

本轮目标非常单一：**新增轻量“视频模式”浏览层。**  
不是做短视频流，不是做自动连播，不是学小红书首页双列瀑布流。重点是：在当前个人朋友圈站里，增加一个安静、克制、适合浏览视频内容的模式。

一、必须完成的内容

1. 扩展顶部模式切换
在当前“全部 / 照片”基础上，增加：
- 全部
- 照片
- 视频

要求：
- 保持当前单页面结构
- 不要改成多页面
- 不要引入复杂路由
- 模式切换逻辑继续放在当前时间流层里处理
- 样式与现有模式切换保持一致或做最小必要扩展，不要顺手重做整套 UI

2. 新增视频模式
在 `Timeline.tsx` 中新增视频模式视图。
视频模式只展示 `post.type === 'video'` 的动态，不展示图片、文章、音乐、纯文字等其他类型。

展示方式要求：
- 不要做双列瀑布流
- 不要做九宫格视频墙
- 使用单列大卡片或明显偏大的视频封面卡片列表
- 每条视频卡至少显示：
  - 视频封面
  - 播放按钮
  - 时长
  - 简短文字内容
  - 发布时间
  - location（如果当前项目已支持 location 且该条视频有 location）
- 整体更像“视频浏览层”，不是“短视频 feed”

3. 视频模式首屏
视频模式不要一切进来就直接一长列普通卡片。
请在视频模式顶部增加一个轻量首屏区域，至少包含：
- 标题：视频
- 一句轻说明，例如“把会动的片段单独留出来”
- 一个较大的首图/首视频封面展示区（可取当前过滤结果中的第一条视频作为 hero）

要求：
- 首屏要轻，不要做夸张视觉演出
- 不要引入复杂动效
- 不要做重毛玻璃或概念稿式样式
- 它应该像一个内容模式的开头，而不是独立产品首页

4. 点击行为
视频模式中的视频点击行为保持克制：
- 优先使用“原位播放”
或
- 进入一个轻量全屏播放层

两种方式任选一种，但要求：
- 不要自动连续播放下一条
- 不要做上下滑切视频
- 不要做短视频沉浸流
- 不要影响当前时间流中的视频播放逻辑，除非为了复用做最小必要整理

5. demo 数据补充
在 `src/services/demoData.ts` 中补充足够的视频 demo，保证视频模式中至少能看到：
- 1 条短视频，偏竖屏封面
- 1 条短视频，偏横屏封面
- 1 条带文字说明的视频
- 1 条带 location 的视频（如果当前项目已支持 location）
- 1 条带 tags 的视频

要求：
- 不要删除现有 demo 数据
- 在现有基础上增加
- 文案简短自然，像真实朋友圈内容
- 发布时间合理分布，保证视频模式列表有浏览价值

二、实现约束

1. 不要新增新的页面
2. 不要改成短视频产品
3. 不要做自动连播
4. 不要做连续滑动切视频
5. 不要引入新的依赖
6. 不要重构评论、点赞、标签、照片模式逻辑
7. 不要重写整个 `MomentCard`
8. 不要为了视频模式顺手大改现有时间流结构
9. 不要做小红书式双列瀑布视频流
10. 不要做复杂动画系统

三、显示要求

1. 在“视频”模式下，只显示视频内容
2. 顶部首屏应明显存在，但不要过高或过重
3. 视频列表应明显区别于照片模式，不能只是把视频封面换个地方摆出来
4. 视频卡片在移动端优先可看，封面尺寸和信息密度要合理
5. 时间流中的其他模式（全部 / 照片）保持原有逻辑不被破坏

四、允许的最小整理

1. 如果为了避免重复代码，需要抽一个很小的视频卡片子组件，可以新增，但必须克制
2. 如果为了视频模式需要在 `Timeline.tsx` 内增加一个 `VideoView` 之类的小视图组件，可以接受
3. 如果为了展示 location / 时间 / 时长需要轻量整理现有视频展示代码，可以做最小改动
4. 不要借机铺大量新文件

五、完成后要求

1. 至少执行一次可验证检查，如 `npm run build`
2. 输出时明确汇报：
- 本轮修改了哪些文件
- 视频模式如何进入
- 视频模式首屏如何展示
- 视频卡片展示了哪些信息
- 点击视频后采用了哪种播放方式
- 新增了多少条视频 demo
- 构建结果

重点提醒：本轮不是做“视频产品”，而是在当前朋友圈站中增加一个安静、克制、可浏览的视频模式。
