基于当前项目实际状态执行本任务。先阅读并理解以下文件后再动手：src/app/App.tsx、src/app/App.module.css、src/components/Timeline.tsx、src/components/MomentCard.tsx、src/components/MomentCard.module.css、src/types/moment.ts，以及与 timeline 状态持久化相关的 util/hook。确认当前项目已经具备 全部 / 照片 / 视频 模式、收藏、归档、标签筛选、模式记忆 后，再执行本任务。

本轮目标不是继续加大功能，而是做一次高级交互层级整理：**顶部减负 + 作者入口后置 + 视频模式轻筛选。**
重点是让当前站点更像长期使用的内容 App，而不是功能逐渐堆满的单页网页。

一、本轮必须完成的内容

1. 顶部减负：重整时间流顶部控制区
当前顶部控制区已经同时承载模式切换、收藏、归档、标签等能力。请在不删核心能力的前提下，做“层级收敛”。

目标：
- 模式切换（全部 / 照片 / 视频）继续保持一级入口地位
- 收藏、归档、标签不应继续全部同权地长期平铺在第一层
- 让顶部第一屏看起来更像内容页，而不是筛选工具条

实现要求：
- 模式切换继续常驻
- 收藏、归档、标签至少要有一部分改成“默认收敛、按需展开”或“只显示当前状态”的形式
- 不要做复杂弹窗系统，但可使用轻量展开区、sheet、折叠区、状态胶囊等方式
- 保持当前功能可达，不要因为“减负”把功能藏到无法发现
- 不要重写整个 Timeline 结构

2. 作者入口后置：把作者模式从封面显性入口改为头像交互入口
当前封面顶部直接展示作者入口，长期看会影响内容页气质。请将作者入口后置到头像交互中。

目标：
- 点击头像后，先出现一个轻量 profile sheet / profile card / 身份浮层
- 该浮层中展示：
  - 头像
  - 昵称
  - 个签
  - 作者模式入口
- 再从该入口进入当前作者登录/作者模式逻辑
- 保留现有作者功能能力，不改变作者认证逻辑本质

要求：
- 不要做复杂用户系统
- 不要做大型弹层系统
- 不要删除作者能力
- 只调整入口方式与交互层级
- 作者入口从“显性管理按钮”改为“身份层中的后置入口”

3. 视频模式轻筛选
当前视频模式已经具备首屏 hero 和视频列表。请在视频模式内部增加一层轻量筛选能力，让它更适合长期浏览。

目标：
- 不改变“全部 / 照片 / 视频”一级模式结构
- 在视频模式内部增加轻量二级浏览能力，例如从以下方向中选择最合适的一种或两种：
  - 最近 / 全部
  - 某标签
  - 某月份
  - 仅收藏的视频
- 重点是“轻”，不要把视频模式做成复杂控制台

要求：
- 复用当前已有的收藏、归档、tag 数据与状态逻辑，优先组合而不是重新发明一套
- 视频模式内部的轻筛选应明显有用，而不是纯样式占位
- 不要做复杂 tabs 套 tabs
- 不要把视频模式变成另一个独立产品

二、设计与交互要求

1. 顶部控制区的设计要求
- 第一视觉重心应回到内容，而不是控制按钮
- 模式切换保持最清晰
- 其他控制能力要更轻、更后置、更低存在感
- 选中状态仍需清晰
- 不要做成后台过滤面板
- 不要增加过多新按钮

2. 作者入口的设计要求
- 头像点击应有明确但克制的反馈
- profile 浮层应更像内容页中的身份层，而不是管理后台入口
- 进入作者模式的按钮/文案可以弱化，但必须可发现
- 保持移动端优先体验
- 不要用 window prompt 一类粗糙实现

3. 视频模式的设计要求
- 视频模式仍然保持“安静的视频浏览层”方向
- 轻筛选不应破坏视频 hero + 列表的主结构
- 轻筛选最好与长期浏览相关，而不是为了功能而功能
- 不要做自动播放流
- 不要做连续切视频
- 不要做小红书式双列瀑布视频 feed

三、实现约束

1. 不要新增页面
2. 不要引入新的依赖
3. 不要重构 PocketBase 接入逻辑
4. 不要大改内容类型模型
5. 不要重写评论、点赞、灯箱、编辑删除等既有功能
6. 不要顺手做复杂动画系统
7. 不要把所有新逻辑糊进单一超长文件
8. 允许少量新增轻量组件或 util，但必须克制

四、允许的合理实现方式

1. 顶部减负可用：
- 状态胶囊 + 展开区
- 轻量折叠区
- 小型 bottom sheet
- 只显示当前 tag / 当前月份，再展开完整列表

2. 作者入口后置可用：
- profile sheet
- profile card
- 头像触发的轻量浮层

3. 视频模式轻筛选可用：
- 轻 segmented control
- 状态胶囊
- 极轻量 filter row
但必须简洁，不要再制造新的复杂工具条

五、完成后要求

1. 至少执行一次可验证检查，如 npm run build
2. 输出时明确汇报：
- 本轮修改了哪些文件
- 顶部控制区具体如何减负
- 哪些能力改成了收敛显示
- 作者入口如何从头像进入
- 视频模式增加了哪种轻筛选
- 当前交互层级相比修改前有什么变化
- 构建结果

重点提醒：本轮不是继续“加功能”，而是整理已有能力的交互层级，让页面更像一个长期可用的内容 App。
