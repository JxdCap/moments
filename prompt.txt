基于当前项目实际状态执行本任务。不要重写项目，不要新开第二套实现，不要推翻现有 React + Vite + TypeScript + CSS Modules + PocketBase 结构；在现有代码基础上做“第三轮 UI / 交互打磨”。本轮只做界面与交互细节修正，不做大规模架构改造。

目标：在当前第二版基础上，进一步提升“微信朋友圈味”，重点打磨操作菜单、点赞区、灯箱沉浸感、评论区视觉。保留当前作者登录入口，不要弱化、隐藏、迁移或重做作者登录区域。

一、本轮必须完成的内容

1. 三个点操作菜单更像微信朋友圈
- 保留当前右下角操作入口，但把样式和交互打磨得更接近官方朋友圈
- 不要继续使用过于直接的纯文本“...”按钮视觉；改成更像朋友圈的操作触发器
- 菜单本体要更紧凑、更轻、更像朋友圈里的赞/评论弹出层
- 菜单位置、尺寸、圆角、阴影、分隔线、文字节奏都尽量贴近朋友圈
- 可以加入很轻的展开/收起过渡，但不要做花哨动画
- 当前菜单项保留：点赞、评论

2. 点赞区更像朋友圈互动块
- 保留公开点赞功能
- 优化点赞区的文案、布局和视觉，使其更像朋友圈里的点赞互动区，而不是普通网页提示语
- 不要使用“觉得不错”这类偏产品化文案
- 点赞区应与评论区统一为一个更像朋友圈互动灰块的视觉系统
- 若当前是匿名公开点赞，可采用更自然的轻量展示方式；重点是视觉与交互成立
- 保留现有 demo / 本地状态逻辑与 PocketBase 对接位，不要擅自改成复杂用户体系

3. 图片灯箱更沉浸、更像朋友圈
- 保留当前全屏黑底预览能力
- 去掉过于网页化、按钮化的明显文字控件风格，尤其是“上一张”“下一张”“关闭”这类强文字按钮感
- 让灯箱更接近朋友圈图片预览的沉浸态：图片更突出，操作控件更弱
- 继续保留：关闭、切换、索引显示、锁定背景滚动
- 若实现成本合适，可加入更自然的点击左右区域切换逻辑；但不要引入第三方 UI 库
- 不必强行做复杂手势，优先把观感做对

4. 评论区视觉再向朋友圈靠拢
- 保留当前“点击评论后展开评论框”的交互，不要改回常驻输入框
- 优化评论区整体视觉，使其更像朋友圈互动块，而不是博客评论区
- 点赞区和评论区应成为同一套浅灰互动容器中的两个部分，视觉语言统一
- 适当弱化评论头像、分隔、边框和表单存在感，让卡片整体更清爽
- 评论输入框仍然在点击“评论”后展开，但展开后的样式也要更贴合朋友圈，而不是后台表单风格

二、明确保留，不要改动的部分
1. 保留当前作者登录入口，不要弱化、隐藏、移位或重做
2. 保留当前单页面结构
3. 保留当前公开点赞能力
4. 保留当前灯箱功能基础
5. 保留当前评论交互方式（通过菜单触发展开评论框）
6. 保留当前 PocketBase + demo 双模式思路

1. 不要新增与本轮无关的新功能
2. 不要重构为多页面
3. 不要引入 Tailwind、shadcn、任何 UI 组件库
4. 不要改成 Next.js、Nuxt、SSR、全栈 CMS
5. 不要再做大规模“收骨架”；本轮重点不是继续压文件数
6. 不要把现有核心结构继续硬塞到单个超长文件里
7. 不要改作者登录逻辑
8. 不要加入复杂动画、复杂手势、复杂状态管理

四、实现要求
1. 优先复用现有组件与样式文件，做增量打磨
2. 样式仍使用原生 CSS / CSS Modules
3. 重点改动应集中在现有的操作菜单、MomentCard、Lightbox、互动区样式等相关文件
4. 视觉方向以“更像微信朋友圈”为第一优先级，而不是通用社交 feed
5. 若需要调整文案、图标、间距、圆角、背景、阴影、按钮样式，可直接修改现有实现
6. 可以轻量整理相关样式归属，但不要为了整洁而大动结构
