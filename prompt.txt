基于当前项目实际状态执行本任务。先阅读并理解以下文件后再动手：src/app/App.tsx、src/app/App.module.css、src/components/OwnerBar.tsx、src/components/Timeline.tsx，以及当前与收藏、归档、标签、模式记忆相关的 util。确认当前项目已经完成以下能力后再执行本任务：顶部筛选收敛、头像触发 profile sheet、视频模式轻筛选。

本轮目标非常单一：**不新增功能，只对这次已完成的高级交互层级整理做微调。**  
重点是让现有交互更自然、更安静、更像长期内容 App，而不是继续堆能力。

一、本轮必须完成的微调内容

1. 顶部筛选层微调
当前顶部已经从“平铺控制条”改成“模式切换 + 筛选入口 + 当前状态摘要”，方向是对的。本轮只做微调，让它更轻、更顺、更像内容页控制层。

要求：
- 保留当前“全部 / 照片 / 视频”常驻一级入口
- 保留当前筛选收敛逻辑，不要改回平铺
- 保留当前 summary chip 状态摘要，不要删除
- 只微调以下方向：
  - “筛选”入口的存在感再轻一点
  - summary chip 与提示文案的层级更清楚
  - 展开后的 filterPanel 节奏更顺，不像后台筛选面板
  - 归档、标签、收藏三块之间的间距、标题、清除按钮关系更自然
- 不要新增新的筛选项
- 不要改筛选逻辑本身
- 不要重写 TimelineControls 结构

2. Profile sheet 微调
当前作者入口已经后置到头像触发的资料层中，方向正确。本轮只微调这个资料层，让它更像身份层，而不是“说明设计逻辑的弹层”。

要求：
- 保留头像触发 profile sheet 的交互
- 保留当前 OwnerBar 继续在资料层中工作
- 不要改作者登录逻辑
- 只微调以下方向：
  - profile card 中说明文案弱化、缩短，减少“解释设计”的感觉
  - 资料层更像自然身份层，而不是管理入口容器
  - 头像、昵称、个签与作者模式区域之间的层级更清楚
  - “关闭”按钮和“作者模式”区域的关系更自然
- 不要新增复杂弹层系统
- 不要加入额外管理按钮
- 不要做大动画

3. 视频模式轻筛选微调
当前视频模式已经加入“最近 / 全部”和“仅收藏”，方向正确。本轮只微调视频模式首屏和筛选区节奏。

要求：
- 保留当前视频模式结构：hero + 列表
- 保留当前“最近 / 全部”
- 保留当前“仅收藏视频 / 全部视频”
- 只微调以下方向：
  - 视频 hero 与筛选区之间的视觉节奏更自然
  - 视频轻筛选区更像浏览辅助层，而不是第二层工具条
  - “最近 / 全部”与“仅收藏”之间的主次更清楚
  - 视频首屏文案与统计文案更简洁自然
  - 如果当前 hero 的信息略重，可做轻量收敛，但不要改掉 hero 结构
- 不要新增视频模式内更多筛选维度
- 不要做自动播放流
- 不要重写 VideoView 整体结构

二、明确不要做的事

1. 不要新增功能
2. 不要新增页面
3. 不要加入新依赖
4. 不要改评论、点赞、灯箱、收藏、归档、模式记忆逻辑
5. 不要改作者认证逻辑
6. 不要把 tag / 归档 / 收藏重新平铺回顶部第一层
7. 不要做复杂动效系统
8. 不要借机重构整个项目
9. 不要顺手大改视觉风格

三、实现原则

1. 本轮以样式、文案、层级、少量 JSX 结构微调为主
2. 优先复用现有 class 和组件
3. 若需要新增少量 class 名，可以新增，但保持克制
4. 当前交互方向已经正确，本轮重点是“收口”和“减噪”
5. 所有调整都应服务于：
- 第一屏更安静
- 控制层更后置
- 身份层更自然
- 视频模式更耐看

1. 至少执行一次可验证检查，如 npm run build
2. 输出时明确汇报：

- 视频模式轻筛选具体做了哪些微调
- 哪些文案被弱化或收短了
- 构建结果

重点提醒：本轮不是继续扩展产品能力，而是打磨已经做对的交互层级，让整体体验更自然、更高级、更耐看。
